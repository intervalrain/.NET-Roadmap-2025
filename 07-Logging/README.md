# Chapter 7: Logging 總結

在本章中，我們探討了現代 .NET 應用程式中至關重要的一環——日誌記錄。我們從 .NET 提供的標準抽象層開始，一直到整合強大的第三方日誌框架。

## 1. Microsoft.Extensions.Logging
我們首先學習了 .NET 內建的日誌抽象函式庫 `Microsoft.Extensions.Logging`。它是實現日誌系統解耦合的關鍵。
- **核心理念**：透過一組通用介面 (`ILogger`, `ILoggerFactory`) 和日誌提供者 (Providers) 的設計，讓應用程式程式碼與具體的日誌實現分離。
- **核心元件**：我們熟悉了 `ILogger` 的使用、六種不同的「日誌級別」(Log Levels)，以及如何設定多個日誌提供者。
- **結構化日誌**：我們了解了「結構化日誌」的核心思想——將訊息範本與參數分開記錄，這極大地增強了日誌的可查詢性和可分析性。

## 2. Serilog
接著，我們學習瞭如何整合 .NET 生態系中最受歡迎的結構化日誌框架——**Serilog**。
- **為何選擇 Serilog**：我們了解到 Serilog 是專為結構化日誌而生，擁有極其豐富的 **Sinks** (輸出目標) 和 **Enrichers** (內容擴充器) 生態。
- **整合與設定**：我們學習瞭如何在 ASP.NET Core 應用程式中無縫整合 Serilog，並透過 `appsettings.json` 進行靈活的設定，實現了程式碼與設定的分離。
- **強大功能**：透過設定範例，我們看到了如何輕鬆地將日誌同時輸出到主控台和滾動檔案，並如何設定過濾規則和輸出範本。

總結來說，本章為你建立了一套現代化的 .NET 日誌知識體系。你學會了利用 `Microsoft.Extensions.Logging` 的抽象來建構可維護的程式碼，並透過整合 Serilog 來實現功能強大、可靈活設定的結構化日誌系統。這項技能將幫助你更有效地監控應用程式、診斷問題，並從日誌中挖掘出有價值的資訊。
